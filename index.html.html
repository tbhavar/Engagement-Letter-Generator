📋 Pre-Workshop Setup (For Participants)

Participants need these in 2 minutes before workshop starts:

  • GitHub Account - Free at github.com
  • Claude.ai Account - Free access for AI-powered code generation
  • Any Browser - Chrome, Firefox, Safari
  • Text Editor - VS Code or Notepad++
✅ That's it! No Python installation, no local server setup, no complexity.
⚠️ Important Limitations:
  • Streamlit Cloud (Community): Only 1 private app per workspace. Use public repos or upgrade to paid tier for multiple private deployments.
  • GitHub Free: Unlimited public/private repos but limited to 3GB storage for LFS files.
  • Streamlit Cloud Free Tier: 1 GB RAM, 1 vCPU. Sufficient for small-medium CA firms.

🎯 PHASE 1: Command Claude to Generate App (30 minutes)

Have participants paste this exact prompt into Claude.ai:

PROMPT FOR CLAUDE: Write a complete Streamlit web app that generates engagement letters for CA firms. Requirements: 1. Login system with password "CA2026" 2. User inputs: Client name, Firm name, Service type (dropdown: Full Audit, Limited Review, Income Tax, GST, Company Formation), Fee amount 3. Generate professional engagement letter using Jinja2 templating 4. Show live preview before saving 5. Download as text file button 6. Help tab with FAQs Constraints: - Use ONLY Streamlit and Jinja2 (no complex dependencies) - Include proper error handling - Add comments for CAs with zero coding knowledge - Make login case-insensitive - Professional styling with gradient header - 5 service types with auto-populated standards Output: Complete, runnable app.py file ready to deploy

Expected Result: Claude generates ~1000 lines of clean Python code

Trainer Demo (Show This Live):

pip install streamlit jinja2 streamlit run app.py

App opens in browser. Participants see it's working!

📝 PHASE 2: Create GitHub Repository (15 minutes)

# Action
1 Go to github.com → Create new PRIVATE repo named "engagement-letters"
2 Copy Claude-generated code → Create file "app.py"
3 Create file "requirements.txt" with content: streamlit and jinja2
4 Create file "README.md" with quick start instructions
5 Create file ".gitignore" to exclude __pycache__
6 Commit all files with message: "Initial engagement letter app"

🚀 PHASE 3: Deploy to Streamlit Cloud (20 minutes)

Step-by-Step Deployment:

  1. Sign up at share.streamlit.io (free tier)
  2. Click "New app" → Select GitHub repo "engagement-letters"
  3. Set Main file to "app.py"
  4. Click Deploy → Get live link in 2-3 minutes
  5. Share link with participants: https://share.streamlit.io/[username]/engagement-letters/app
✅ RESULT: Live web app accessible from anywhere on any device!

For participants wanting to master the entire development cycle in one IDE: Use Antigravity IDE to vibe-code, version-control, and deploy—all from one interface.

Prerequisites (5 minutes):

  • Git installed on your machine
  • GitHub extension installed in Antigravity IDE
  • GitHub Personal Access Token (PAT) configured in Antigravity IDE
  • Private GitHub repository created (from Phase 2)

👨‍🏫 Trainer Tips & Teaching Philosophy

Key Messages for Participants:

  • This is NOT "Learn Python" — it's "Learn to Command AI"
  • You don't write code — you guide an AI that writes code
  • English, not syntax — describe what you want in plain English
  • This is real business skill — used today in top firms
  • Zero infrastructure cost — Streamlit Cloud free tier + GitHub free

How to Teach It:

✅ Show, don't tell: Live demo Claude generating code in real-time (2 minutes)
✅ Emphasize the "vibe": "You speak English. AI writes Python. You guide. Done."
✅ Test together: Generate, deploy, test the live web app as a group (show it works!)
✅ Iterate with Claude: Ask "Add a feature" or "Fix this bug" and regenerate (5 minutes)
✅ Measure impact: "15 minutes → 3 minutes per letter = 80% time savings"
✅ Business focus: This is agentic workflows for CA firms, not a coding course

🔧 Troubleshooting

Issue Fix
Import Error Ask Claude: "Fix import errors and add missing libraries to requirements.txt"
Deployment Fails Check: requirements.txt exists, repo is PRIVATE, GitHub is connected
Code Not Working Share error message with Claude: "Debug this error: [error text]"
App Too Slow Ask Claude: "Optimize performance using st.cache_resource"
GitHub Integration Failing Verify GitHub token is in Streamlit Cloud secrets (Settings → Secrets)
Can't Access Streamlit Link Check: repo is public, app is deployed (check Streamlit Cloud dashboard)
